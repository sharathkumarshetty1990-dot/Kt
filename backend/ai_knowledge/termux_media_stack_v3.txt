Termux Media Stack Manual v3
Expanded with when-to-use guidance, dependency notes, pairings, and 10 example user instructions per
package.
Scope note: packages that appear in both Termux and Python sections are kept as separate entries so the installation path
stays clear.
Video editing

ffmpeg + libass + libx264/libx265 + libvpx/libaom + moviepy/editly

Audio cleanup

sox + rubberband + libsoxr/libsamplerate + librosa/scipy

OCR

tesseract + leptonica + pytesseract/rapidocr + Pillow/OpenCV

Subtitle work

libass + pysubs2 + pycaption + srt + fontconfig/harfbuzz/fribidi

AI media automation

ffmpeg-python + ImageIO + yt-dlp + numpy + OpenCV + transcripts

How to read this guide
Each package entry includes a short purpose line, a when-to-use note, a dependency note, a pairing note, and ten short
example user instructions. The four target workflows are video editing, audio cleanup, OCR, subtitle work, and AI media
automation.

alsa-lib
Workflow focus: audio cleanup
Purpose: Low-level audio interface library.
When to use: Use this when playback, capture, cleanup, routing, or DSP support is needed.
Depends on: Usually depends on the surrounding media toolchain and a compatible front end.
Pairs well with: The rest of the media toolchain for end-to-end workflows.
10 example user instructions:
1. Use alsa-lib to clean, resample, or normalize audio.
2. Use alsa-lib when a waveform or soundtrack sounds uneven.
3. Use alsa-lib to convert between lossless and compressed audio formats.
4. Use alsa-lib to trim silence, fade, or smooth audio clips.
5. Use alsa-lib when you need better playback or recording support.
6. Use alsa-lib to reduce noise, echo, or rough edges in speech.
7. Use alsa-lib for batch audio repair in a media workflow.
8. Use alsa-lib with sox, librosa, scipy, or ffmpeg.
9. Use alsa-lib when your project is voice cleanup or sound analysis.
10. Use alsa-lib whenever the audio step is the bottleneck.
