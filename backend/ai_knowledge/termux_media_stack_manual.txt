                      Termux Media Stack Usage Manual
      Practical guidance for video editing, image processing, audio, OCR, Python, and npm tools

Scope: This document gives a practical usage guide for every package in your list. Many entries are
libraries, not standalone commands, so their 'usage' is usually through FFmpeg, Python, or another
app that depends on them.



Common workflow patterns
雞ｷ   Trim and transcode video: ffmpeg -i input.mp4 -ss 00:00:05 -to 00:00:20 -c:v libx264 -crf 23
    -preset medium -c:a aac output.mp4
雞ｷ   Extract audio only: ffmpeg -i input.mp4 -vn -c:a libmp3lame output.mp3
雞ｷ   Burn subtitles: ffmpeg -i input.mp4 -vf "subtitles=subs.srt" output.mp4
雞ｷ   Normalize audio: sox input.wav output.wav gain -n -3
雞ｷ   OCR from a screenshot: tesseract image.png stdout



Package reference
Tip: for libraries, install them when a tool asks for them or when a script fails with a missing
dependency. They are the gears inside the machine, not usually the handle you grab directly.



Video, codecs, and containers
雞ｷ   ffmpeg: Main multimedia workhorse. Convert, trim, join, filter, encode, mux, demux, and burn
    subtitles. Typical patterns: ffmpeg -i input.mp4 -c:v libx264 -crf 23 -preset medium out.mp4;
    ffmpeg -i in.mp4 -vn audio.m4a; ffmpeg -i in.mp4 -vf "scale=1280:-2,fps=30" out.mp4.

雞ｷ   libavif: AVIF image format library.

Best-practice notes
雞ｷ   Use FFmpeg as the central hub and let codecs and helper libraries plug into it.
雞ｷ   Use Python packages when you need repeatable automation, subtitle handling, analysis, or batch
    processing.
雞ｷ   Use npm globals only for CLI tools that specifically help with scripted video assembly.
雞ｷ   Check each package's own --help, README, or man page for flags after installation.
