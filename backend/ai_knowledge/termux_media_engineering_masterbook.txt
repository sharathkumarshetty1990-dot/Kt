Termux Media Engineering Masterbook
The missing layer: editing logic, workflow design, dependency thinking, and recipe maps
Built for video editing, audio cleanup, OCR, subtitle work, and AI media automation.

How to use this book: read the workflow chapters first, then jump to the package map and the recipe
appendix. This book is meant to teach decision-making, not just memorizing package names.
Scope note: package availability and exact build behavior can vary across Termux repositories and device
hardware. Treat this as a practical field guide, then verify on your own device with small test clips.

1. The editing mindset
Every media task has the same skeleton:
ď‚· Ingest - bring media into the pipeline.
ď‚· Normalize - resize, reformat, resample, rename, or convert into a clean working format.
ď‚· Process - edit, analyze, enhance, annotate, transcribe, or transform.
ď‚· Validate - inspect output visually and audibly before trusting it.
ď‚· Export - write the final file in a format that matches the target platform.
The big mistake is to start with tools instead of outcomes. First decide the job: cut a video, clean a noisy
track, read text from frames, make subtitles, or automate a batch workflow. Then choose the smallest
package chain that can do the job reliably.
In practice, most powerful workflows in Termux are built from a few anchors: ffmpeg for media I/O and
filtering, Python for glue, OpenCV for visual analysis, and one specialist package for the jobâ€™s final step.

2. Workflow map
Video editing: Use ffmpeg or ffmpeg-python for trims, transcodes, filters, overlays, speed changes, muxing,
and delivery exports. Add moviepy or editly for scriptable timeline-style edits. Use OpenCV for frame
analysis, detection, and preprocessing.
Audio cleanup: Use ffmpeg, sox, rubberband, librosa, scipy, sndfile, and soundfile-style tooling for denoise,
loudness, resampling, tempo shifts, and segment extraction.
OCR: Use tesseract, leptonica, pytesseract, rapidocr, OpenCV, Pillow, and image conversion tools to extract
text from stills, screenshots, and video frames.
Subtitle work: Use pysubs2, pycaption, srt, font packages, and ffmpeg/libass for create, convert, sync, burn,
style, and embed subtitle flows.
AI media automation: Use Python with numpy, opencv, moviepy, yt-dlp, ffmpeg-python, vidgear, and
transcript tools to automate batch media pipelines.

3. Video editing workflow
Best-fit package chain:
ď‚· ffmpeg - the core engine for trim, crop, scale, concat, transcode, mux, demux, overlay, subtitles, and
filters.
ď‚· ffmpeg-python - helpful when you want Python code to generate complex ffmpeg commands safely.
ď‚· moviepy - useful for quick scripted edits, compositing, and timeline-style automation.
ď‚· editly - useful for structured sequence-based video creation and templated edits.
ď‚· opencv-python - useful for scene analysis, tracking, face detection, keyframe logic, and frame-by-frame
processing.
ď‚· libass, fontconfig, freetype, fribidi, harfbuzz - important for subtitle rendering and correct text shaping.
When to use this stack:
ď‚· Use ffmpeg when the job touches codecs, containers, filters, audio streams, subtitles, or delivery
exports.
ď‚· Use OpenCV when the job depends on what is inside the frames, not just the file wrapper.
ď‚· Use moviepy when the edit is small, scripted, and easier to express in Python than in raw ffmpeg syntax.
ď‚· Use editly when you want reusable clip templates, titles, and simple narrative assembly.
Common editing moves:
ď‚· Trim dead sections and export a clean cut.
ď‚· Crop or scale for social formats like 9:16, 1:1, or 16:9.
ď‚· Add text, logos, subtitles, or picture-in-picture overlays.
ď‚· Normalize audio and make dialogue readable.
ď‚· Blend clips with transitions or cuts.
ď‚· Burn subtitles for platforms that do not accept sidecar files.

Video cautions

ď‚·
ď‚·

4. Audio cleanup workflow
Best-fit package chain:
ď‚· ffmpeg - trim, resample, normalize, convert formats, and extract audio.
ď‚· sox - surgical audio editing, fades, trims, channel ops, and DSP-style processing.
ď‚· rubberband - time-stretch and pitch-shift with decent quality.
ď‚· librosa - analysis, beat detection, onset detection, tempo, chroma, and feature extraction.
ď‚· scipy - filters and signal processing helpers.
ď‚· sndfile / libsndfile - read and write audio accurately.
ď‚· pulseaudio or jack - for live routing or capture in advanced setups.
When to use this stack:
ď‚· Use ffmpeg for most format conversion and simple cleanup.
ď‚· Use sox when you need precise edits, fades, and repeatable command-line audio surgery.
ď‚· Use librosa when your task is analytical, like finding beats, silence, or speech segments.
ď‚· Use rubberband when the user wants time-stretching or pitch changes without making the audio sound
like a melted cassette.
Typical audio tasks:
ď‚· Remove silence, hum, or clipping trouble.
ď‚· Normalize volume across an entire project.
ď‚· Cut a podcast into clips.
ď‚· Extract audio from a video for separate cleanup.
ď‚· Match audio duration to edited video.

Audio cautions
ď‚·
ď‚·
ď‚·

Always compare before and after at the same loudness.
Tempo changes can create artifacts if pushed too far.
For speech, noise reduction should protect consonants, not just lower the noise floor.

5. OCR workflow
Best-fit package chain:
ď‚· tesseract - the main OCR engine.
ď‚· leptonica - image preprocessing support for OCR.
ď‚· pytesseract - Python bridge to tesseract.
ď‚· rapidocr - an alternative OCR route when you want fast automated extraction.
ď‚· opencv-python - deskewing, thresholding, cropping, and region detection.
ď‚· Pillow / imageio / graphicsmagick / openjpeg tools - image conversion and cleanup helpers.
When to use this stack:
ď‚· Use OCR when the content exists as pixels, not as text metadata.
ď‚· Use OpenCV before OCR if the image is noisy, skewed, low contrast, or needs cropping.
ď‚· Use rapidocr or tesseract depending on language, quality, and speed needs.
ď‚· Use ffmpeg to sample frames from video before OCR.
OCR workflow recipe:
1. Extract frames from video or capture screenshots.
2. Crop the text region.
3. Deskew and increase contrast.
4. Run OCR.
5. Clean the text output with scripts or regex.
6. Verify the result against the original image.

OCR cautions
ď‚·
ď‚·
ď‚·

Small text needs good preprocessing more than brute force OCR.
Language packs matter.
If the source is a video, choose keyframes wisely or you will OCR the same visual noise again and
again.

6. Subtitle workflow
Best-fit package chain:
ď‚· pysubs2 - read, edit, retime, and write subtitle files.
ď‚· srt - lightweight subtitle parsing and creation.
ď‚· pycaption - format conversion and caption handling.
ď‚· youtube-transcript-api - transcript retrieval when the source supports it.
ď‚· ffmpeg - mux, burn, or extract subtitle streams.
ď‚· libass, fontconfig, freetype, fribidi, harfbuzz - rendering and text shaping support.
When to use this stack:
ď‚· Use pysubs2 for direct subtitle file editing and syncing.
ď‚· Use srt when you want a simple Python workflow and minimal overhead.
ď‚· Use pycaption when the job is about converting between caption formats.
ď‚· Use ffmpeg + libass when subtitles must be burned into the picture.
Useful subtitle tasks:
ď‚· Shift timing after an edit.
ď‚· Convert between SRT, ASS, VTT, and related formats.
ď‚· Add styling, speaker colors, or karaoke-like effects.
ď‚· Embed subtitles as soft subtitles in a video container.
ď‚· Burn them permanently when platform rules require it.

Subtitle cautions
ď‚·
ď‚·
ď‚·

Font choice can change the final look a lot.
Right-to-left and complex-script rendering depends on shaping libraries.
Subtitle timing should be checked against the final cut, not the rough draft.

7. AI media automation workflow
Best-fit package chain:
ď‚· python + numpy + scipy - glue and numeric work.
ď‚· opencv-python - frame analysis, detection, and batch visual preprocessing.
ď‚· ffmpeg-python - command generation from Python.
ď‚· moviepy - quick scripted edits and clip assembly.
ď‚· yt-dlp - download media for lawful offline processing when appropriate.
ď‚· vidgear - video capture and streaming helpers.
ď‚· imageio-ffmpeg - bridges image pipelines and FFmpeg execution.
When to use this stack:
ď‚· Use this when the user wants a repeated pipeline, not a one-off edit.
ď‚· Use automation for batch file conversion, highlight detection, clip generation, OCR on many frames, or
subtitle generation.
ď‚· Use Python when conditional logic matters more than a single shell command.
Example automation jobs:
ď‚· Scan a folder of clips and produce vertical shorts.
ď‚· Detect scenes and cut highlights into separate files.
ď‚· Extract audio, clean it, and reattach it.
ď‚· Pull text from screenshots in bulk and save structured output.
ď‚· Generate subtitles, then burn or mux them depending on target.

9. Decision trees
Need to cut, crop, convert, or burn subtitles? Start with ffmpeg.
Need to inspect frames or detect what is inside the image? Start with OpenCV.
Need to read text from pictures or frame grabs? Start with OCR tools plus preprocessing.
Need to sync or stylize subtitles? Start with pysubs2 or pycaption, then finish with ffmpeg if rendering is
needed.
Need repeated edits across many files? Build a Python automation pipeline.
